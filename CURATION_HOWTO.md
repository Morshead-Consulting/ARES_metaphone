# Curation report — how to read and use it

## What the report is

`curation_report.txt` is generated by `altspell_gen.py` and lists English words
that are phonetically confusable with each target military acronym. It is an
**assist to expert curation, not a finished product** — the algorithm proposes
candidates; a wargame-experienced reviewer keeps the plausible ones, discards
the noise, and adds real mis-recognitions the algorithm misses.

Since Phase 2 the matching runs on how each acronym is **spoken**, not on its
spelling. Each target block lists the spoken forms used (ISTAR is matched as
"eye star", "is tar", "eye ess tee ay are", ...) and each proposal shows which
spoken form produced it. The authoritative design rationale is in
`ARES_DN_PHON_001.md` §3, with the segmentation approach in `CHANGES_phase2.md`.

---

## How to regenerate it

```
uv run python altspell_gen.py --targets data/targets.txt --use-wordlist --max-key-dist 0 --max-per-target 15 --report --output-dir data/reports
```

The `--output-dir` flag writes the output to a timestamped file in the given
directory instead of stdout. The filename is derived from the targets file and
the current time, e.g. `targets_20260610_143022_curation_report.txt`, so runs
from different inputs are kept distinct. The path is echoed to stderr.

Omit `--output-dir` to write to stdout as before (redirect with `>`):

```
uv run python altspell_gen.py --targets data/targets.txt --use-wordlist --max-key-dist 0 --max-per-target 15 --report > data/curation_report.txt
```

Install `wordfreq` (`pip install wordfreq` / `uv sync`) to rank proposals
common-word-first and show a Zipf score — strongly recommended, as word
commonness is the main thing you are judging.

---

## Report format

```
TARGET  spoken as: "form 1" | "form 2" | ...
    <- candidate_word          key_dist=N surface_dist=N zipf=N.N  via "spoken form"
```

| Field | Meaning |
|---|---|
| `TARGET` | The military acronym from `targets.txt` |
| `spoken as:` | The k-best spoken forms of the acronym used for matching (the full letter-by-letter spell-out is always included as a fallback) |
| `candidate_word` | An English word whose phonetic key is identical or near-identical to one of the target's spoken forms |
| `key_dist` | Levenshtein distance between the two phonetic keys (0 = identical sound; 1 = one-edit near-match) |
| `surface_dist` | Levenshtein distance between the raw strings — a rough guide to how different they look |
| `zipf` | Word-frequency score, if `wordfreq` is installed (>=3.5 common, <2 obscure, 0 = not a known word) |
| `via "..."` | The spoken form that produced the match — shown when it differs from the raw target or candidate (e.g. `via "see bee are en"`) |

Proposals are sorted by phonetic distance, then by word frequency (common
first), then by surface closeness. The list is capped per target
(`--max-per-target 15`).

---

## How to curate

Go through each target block and ask: **could an ASR engine plausibly output this
word instead of the acronym in a military-comms context?**

**Keep** a candidate if:
- It sounds like the acronym when spoken aloud, AND
- It is a word an ASR language model might plausibly predict (a high `zipf` is
  a strong signal here).

**Discard** a candidate if:
- It is an obscure dictionary word unlikely to appear in ASR output (`zipf`
  near 0, or flagged "non-word" / "too obscure").
- It only matched through the full **spell-out** fallback and is not a
  realistic mis-recognition. Word-like acronyms (e.g. CASEVAC) occasionally
  attract odd spell-out matches such as "sisyphus" `via "see ay ess ee vee ay
  see"` — discard these on sight.
- It is implausible in the operational context even if phonetically close.

**Add manually** any real mis-recognitions observed in Sprint 3 recordings that
the algorithm did not surface. Segmentation now gives a useful starting set
even for letter-by-letter acronyms (CBRN -> sabrina, siberian, seaborne; IED ->
its spoken-form matches), but observed errors from real recordings remain the
ground truth and should always be added.

> **Short-acronym caution:** very short spell-outs (IED, SAM) have coarse
> phonetic keys that can attract short high-frequency words ("out", "what").
> Lean on the `zipf` ranking and your operational judgement, and consider
> `--max-surface-dist` to trim the noise.

## Output: the CTC-WS context file

Once curated, drop the `--report` flag to emit lines in the format consumed by
the CTC-WS context-biasing layer:

```
uv run python altspell_gen.py --targets data/targets.txt --use-wordlist --max-key-dist 0 --max-per-target 15 --output-dir data/reports
```

Without `--report` the auto-named file uses the suffix `_ctcws.txt`, e.g.
`targets_20260610_143022_ctcws.txt`.

Each output line has the form:

```
TARGET_spelling1_spelling2_...
```

Paste the curated lines into the CTC-WS context file, replacing or extending
the existing hand-authored entries.
