# Curation report — how to read and use it

## What the report is

The curation report is generated by `altspell_gen.py` and lists English words
that are phonetically confusable with each target military term. It is an
**assist to expert curation, not a finished product** — the algorithm proposes
candidates; a wargame-experienced reviewer keeps the plausible ones, discards
the noise, and adds real mis-recognitions the algorithm misses.

The authoritative design rationale is in `ARES_DN_PHON_001.md` §3.

## Repository layout

| Directory | Contents |
|---|---|
| `data/` | Inputs to the tool: target lists (`targets.txt`, `targets_220626.txt`) and reviewer-supplied candidates (`candidates.txt`) |
| `results/` | Tool-generated reports, named by targets file and timestamp |
| `curated/` | Human work products: annotated reports and the CTC-WS deliverable |
| `docs/` | Documentation (this file) |

## How to regenerate the curation report

```
uv run python altspell_gen.py --targets data\targets.txt --use-wordlist --max-per-target 15 --report --output-dir results
```

`--output-dir` writes a timestamped file to `results\` instead of printing to
the terminal. The filename is derived from the targets file and the current
time, e.g. `targets_20260610_143022_curation_report.txt`, so runs from
different inputs are kept distinct. The path is echoed to stderr.

To print to the terminal instead (useful for a quick check):

```
uv run python altspell_gen.py --targets data\targets.txt --use-wordlist --max-per-target 15 --report
```

## Report format

```
TARGET  [PHONETIC-KEY]  spoken: "spoken form"
    <- candidate_word          key_dist=N surface_dist=N [z=N.N]  via "spoken form"
```

| Field | Meaning |
|---|---|
| `TARGET` | The military term from `targets.txt` |
| `[PHONETIC-KEY]` | Its Double Metaphone code — the sound fingerprint used for matching |
| `spoken:` | The spoken form(s) the tool derived (e.g. "eye star" for ISTAR) |
| `candidate_word` | An English word that sounds like the target's spoken form |
| `key_dist` | Levenshtein distance between phonetic keys (0 = identical sound; 1 = near-match) |
| `surface_dist` | Levenshtein distance between raw strings — a guide to how different they look |
| `z=N.N` | Zipf frequency score (≥3.5 common; <2 obscure; 0 not a known word). Shown if `wordfreq` is installed. |
| `via "..."` | Which spoken form produced the match |

Candidates are sorted by `(key_dist, surface_dist)` — closest phonetic matches first.
The list is capped at 15 per target (`--max-per-target 15`).

## How to curate

Open the report from `results\` and go through each target block. For each
candidate, ask: **could an ASR engine plausibly output this word instead of the
military term in a comms context?**

**Keep** a candidate if:
- It sounds like the term when spoken aloud, AND
- It is a word an ASR language model might plausibly predict (common words, z≥3, are the priority).

**Discard** a candidate if:
- It is an obscure dictionary word unlikely to appear in ASR output.
- It is implausible in the operational context even if phonetically close.

**Add manually** any real mis-recognitions observed in Sprint 3 recordings that
the algorithm did not surface — especially for letter-by-letter acronyms (IED,
CBRN, ATGM) where general dictionary matching is weakest.

Save your annotated report to `curated\` for reference.

---

## Output: the CTC-WS context file

Once the curation report has been reviewed, drop `--report` to emit lines in
the format consumed by the CTC-WS context-biasing layer:

```
uv run python altspell_gen.py --targets data\targets.txt --use-wordlist --max-per-target 15 --output-dir results
```

Without `--report` the auto-named file uses the suffix `_ctcws.txt`, e.g.
`targets_20260610_143022_ctcws.txt`.

Each output line has the form:

```
TARGET_spelling1_spelling2_...
```

Review the generated file against your annotated curation report, remove any
candidates you marked as discarded, and save the result to `curated\ctcws_draft.txt`.
Paste the curated lines into the CTC-WS context file, replacing or extending
the existing hand-authored entries.
